# Keybase Book 中文镜像站术语对照表

为了确保翻译的“信、达、雅”以及技术准确性，请遵循以下术语规范。

## 保留原文 (不翻译)
以下术语在技术社区中通用，强行翻译反而造成困扰，建议保留英文或仅在首次出现时备注中文。

*   **Keybase** (产品名)
*   **Sigchain** (可译为“签名链”，但建议保留或双语对照)
*   **KBFS** (Keybase File System)
*   **GPG / PGP**
*   **Paper Key** (虽然可译为“纸质密钥”，但应用内通常指代特定功能，建议保留或加注)
*   **Stellar** (加密货币名)
*   **Team** (在特定上下文中指 Keybase Team 功能)
*   **Identity** (身份)
*   **Proof** (验证/证明，如 Social Proof)

## 建议翻译
*   **Chapters** -> **章节**
*   **Appendices** -> **附录**
*   **Docs** -> **文档**
*   **Guides** -> **指南**
*   **Next** -> **下一章**
*   **Encryption** -> **加密**
*   **End-to-end** -> **端到端**
*   **Device** -> **设备**
*   **Provision** -> **配置/添加(设备)** (视语境而定)
*   **Revoke** -> **吊销/移除**
*   **Account** -> **账户**
*   **Following** -> **关注**
*   **Follower** -> **关注者**
*   **Block** -> **屏蔽**
*   **Wallet** -> **钱包**
*   **Sites** -> **站点**
*   **Chat** -> **聊天** (作为功能模块名时)
*   **Files** -> **文件** (作为功能模块名时)
*   **Public-key cryptography** -> **公钥加密**
*   **Two-factor authentication** -> **双因素认证**
*   **Exploding messages** -> **阅后即焚**
*   **Ephemeral keys** -> **临时密钥**
*   **Restricted bots** -> **受限机器人**
*   **Trust on first use (TOFU)** -> **首次使用信任**
*   **Commitment scheme** -> **承诺方案**
*   **Live Location** -> **实时位置**


## 翻译原则
1.  **代码块与路径**：严禁翻译代码块、命令行指令、文件路径、API 端点。
2.  **代码注释**：代码示例中的注释**必须**翻译，以帮助理解。
3.  **链接**：保留原始链接，除非链接目标已有对应的中文翻译页面。
4.  **风格**：保持客观、技术性的语气，避免过于口语化。
